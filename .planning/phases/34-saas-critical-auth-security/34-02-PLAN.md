# Plan 34-02: Password Reset / Forgot Password

## Goal

Add a complete password reset flow so users who forget their password can self-recover. Currently there is no "Forgot password?" link and no recovery flow.

## Tasks

### Task 1: Add "Forgot password?" link to login page
- Edit `src/app/(auth)/login/page.tsx`
- Add a link below the password field: "Forgot your password?" -> `/forgot-password`
- Style consistently with existing auth page design (gold accent, dark theme)

### Task 2: Create forgot-password page
- Create `src/app/(auth)/forgot-password/page.tsx`
- Email input form with submit button
- On submit: call `supabase.auth.resetPasswordForEmail(email, { redirectTo: '{origin}/api/auth/callback?type=recovery' })`
- Show success message: "Check your email for a password reset link"
- Show error state if email fails
- Match existing auth page layout (centered card, branding)

### Task 3: Create reset-password page
- Create `src/app/(auth)/reset-password/page.tsx`
- New password + confirm password form
- On submit: call `supabase.auth.updateUser({ password })`
- On success: redirect to `/login` with success toast/message
- Validate: password min 8 chars, passwords match

### Task 4: Update auth callback to handle recovery tokens
- Edit `src/app/api/auth/callback/route.ts`
- Detect `type=recovery` in the URL or token exchange
- Redirect recovery tokens to `/reset-password` instead of the default dashboard

### Task 5: Add public routes to middleware
- Edit `src/middleware.ts`
- Add `/forgot-password` and `/reset-password` to `PUBLIC_ROUTES` array

## Verification
- [ ] "Forgot password?" link visible on login page
- [ ] Forgot password page sends recovery email
- [ ] Clicking email link lands on reset-password page
- [ ] Setting new password works and redirects to login
- [ ] `/forgot-password` and `/reset-password` accessible without auth
