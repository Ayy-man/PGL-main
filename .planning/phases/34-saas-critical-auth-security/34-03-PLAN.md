# Plan 34-03: User Profile / Account Settings Page

## Goal

Give users a way to view and update their own account details (name, password) after onboarding. Currently there is no profile or settings page.

## Tasks

### Task 1: Create profile server action
- Create `src/app/actions/profile.ts`
- `updateProfile(formData)` — updates display name in `public.users` table
- `changePassword(formData)` — calls `supabase.auth.updateUser({ password })` for the current user
- Validate inputs server-side (name not empty, password min 8 chars)
- Return success/error responses

### Task 2: Create settings page
- Create `src/app/[orgId]/settings/page.tsx`
- **Profile section**: Display name (editable input), email (read-only display)
- **Password section**: Current password (optional, for confirmation UX), new password, confirm password
- **Account info section**: Role (read-only badge), tenant name (read-only), member since date
- Use existing Card/CardHeader/CardContent components
- Wire forms to server actions from Task 1
- Show success/error toasts on save

### Task 3: Add Settings to sidebar navigation
- Edit `src/components/layout/nav-items.tsx` (or equivalent nav config file)
- Add "Settings" nav item with Settings icon from lucide-react
- Available to all roles (super_admin sees it in admin context, tenant users in org context)
- Route to `/{orgId}/settings`

## Verification
- [ ] Settings page loads for all roles
- [ ] Name change persists and shows in sidebar/team page
- [ ] Password change works (logout + login with new password succeeds)
- [ ] Email and role are displayed but not editable
- [ ] Settings link appears in sidebar for all roles
