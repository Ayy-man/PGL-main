# Plan 34-05: Session/Token Expiry Handling

## Goal

Handle expired Supabase JWT sessions gracefully. Currently, if a user's token expires mid-session, client-side fetches fail silently or show raw errors. Add a global auth state listener that redirects to login on SIGNED_OUT events.

## Tasks

### Task 1: Create SessionGuard component
- Create `src/components/auth/session-guard.tsx` as a client component (`"use client"`)
- Use `useEffect` to set up `supabase.auth.onAuthStateChange` listener
- On `SIGNED_OUT` event: redirect to `/login` via `router.push('/login')`
- On `TOKEN_REFRESHED` event: no action needed (Supabase handles this)
- Clean up subscription on unmount
- Render `null` (no visible UI)

### Task 2: Mount in root layout
- Edit `src/app/layout.tsx`
- Import and include `<SessionGuard />` inside the body, after providers
- Ensure it's inside the Supabase provider context so it can access the client

## Verification
- [ ] Session expires (or manually clear auth cookie) -> user redirected to /login
- [ ] No raw error pages shown on expired session
- [ ] Normal navigation works when session is valid
- [ ] Token refresh works transparently (no redirect on refresh)
