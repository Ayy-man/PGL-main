<objective>
Enhance the Exa API data pipeline to request and surface richer result data (highlights, summary, author, image, favicon), extend the ScrapbookCard type with optional Exa-sourced fields, and upgrade the digest prompt to use Exa's LLM-generated summary/highlights as higher-signal input.

Purpose: The research panel currently only fetches raw text from Exa. Exa already runs LLM processing server-side to generate highlights (with cosine similarity scores), page summaries, author info, and favicons. Requesting these fields adds no latency (same API call) but provides dramatically richer data for both the digest LLM and the UI.

Output: Updated type contracts, enhanced Exa fetch, smarter digest prompt, route wiring for new fields.
</objective>
