**File 1: `src/lib/search/channels/exa-channel.ts`**

Create the Exa channel adapter. This does NOT reuse enrichExa directly (that function returns ExaResult for enrichment, not ChannelResult). Instead, replicate the Exa API call pattern optimized for research queries.

1. Import: `ChannelParams`, `ChannelOutput`, `ChannelResult`, `registerChannel` from `./index`, `getChannelCache`, `setChannelCache` from `../channel-cache`, `withCircuitBreaker` from `@/lib/circuit-breaker`, `trackApiUsage` from `@/lib/enrichment/track-api-usage`.

2. Implement `searchExaInternal(params: ChannelParams): Promise<ChannelOutput>`:
   - Record `startMs = Date.now()`
   - Check cache: `const cached = await getChannelCache("exa", params.query, params.prospect.id, params.tenantId)`. If found, return `{ ...cached, cached: true, latencyMs: Date.now() - startMs }`.
   - Check `process.env.EXA_API_KEY`. If missing, return `{ channelId: "exa", results: [], cached: false, latencyMs: 0, error: "EXA_API_KEY not configured" }`.
   - Build search query: use `params.query` directly (the intent classifier has already reformulated it).
   - POST to `https://api.exa.ai/search` with:
     - Headers: `{ "x-api-key": apiKey, "Content-Type": "application/json" }`
     - Body: `{ query: params.query, type: "auto", numResults: 10, contents: { text: { maxCharacters: 500 } } }`
   - Track usage: `trackApiUsage("exa").catch(() => {})` (fire-and-forget)
   - Map API response `results[]` to `ChannelResult[]`:
     - `channelId: "exa"`
     - `headline: result.title || "Untitled"`
     - `summary: (result.text || "").slice(0, 300)`
     - `source_url: result.url`
     - `source_name: "Exa"`
     - `event_date: result.publishedDate || null`
     - `category: "media"` (default; Exa returns general web content)
     - `relevance: "medium"` (default; orchestrator re-ranks later)
     - `raw_snippet: result.text || ""`
   - Build output, cache it via `setChannelCache`, return it.
   - Wrap entire body in try/catch. On error: return `{ channelId: "exa", results: [], cached: false, latencyMs: Date.now() - startMs, error: String(error) }`.

3. Export `searchExa` via `withCircuitBreaker(searchExaInternal, { name: "exa-channel", timeout: 10000 })`.

4. Register: `registerChannel("exa", searchExa)`.

**File 2: `src/lib/search/channels/edgar-efts-channel.ts`**

Create the EDGAR EFTS full-text search channel. This is a NEW channel distinct from the existing `enrichEdgar` (which uses CIK-based submissions lookup). EFTS searches filing content by keywords.

1. Import: `ChannelParams`, `ChannelOutput`, `ChannelResult`, `registerChannel` from `./index`, `getChannelCache`, `setChannelCache` from `../channel-cache`, `withCircuitBreaker` from `@/lib/circuit-breaker`, `edgarRateLimiter` from `@/lib/rate-limit/limiters`.

2. IMPORTANT: Both edgar-efts AND the existing enrichEdgar share the same `edgarRateLimiter` (10 req/s at `sec-edgar:global` key). Do NOT create a separate limiter.

3. Implement `searchEdgarEftsInternal(params: ChannelParams): Promise<ChannelOutput>`:
   - Record `startMs = Date.now()`
   - Check cache first via `getChannelCache("edgar-efts", ...)`
   - Check `process.env.SEC_EDGAR_USER_AGENT`. If missing, return empty with error.
   - Rate limit: `await edgarRateLimiter.limit("sec-edgar:global")` — if `!result.success`, wait `Math.max(result.reset - Date.now(), 100)` ms then retry once.
   - Build EFTS query: `const searchTerms = \`"${params.prospect.full_name}" "${params.prospect.company || ''}"\`.trim()`
   - Calculate `oneYearAgo` as ISO date string (YYYY-MM-DD)
   - Fetch: `GET https://efts.sec.gov/LATEST/search-index?q=${encodeURIComponent(searchTerms)}&forms=4,13F,8-K,10-K,10-Q&dateRange=custom&startdt=${oneYearAgo}&_source=period_of_report,file_date,entity_name,form_type`
   - Headers: `{ "User-Agent": process.env.SEC_EDGAR_USER_AGENT!, "Accept": "application/json" }`
   - Parse response JSON. Expected shape: `{ hits: { hits: Array<{ _source: { file_date, entity_name, form_type, period_of_report } }>, total: { value: number } } }`
   - Map hits to `ChannelResult[]` (limit to first 10):
     - `channelId: "edgar-efts"`
     - `headline: "${hit._source.form_type} Filing — ${hit._source.entity_name}"`
     - `summary: "Filed on ${hit._source.file_date}. Period: ${hit._source.period_of_report || 'N/A'}."`
     - `source_url: "https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany&company=${encodeURIComponent(params.prospect.company || '')}&type=${hit._source.form_type}&dateb=&owner=include&count=10"` (best-effort link)
     - `source_name: "SEC EDGAR"`
     - `event_date: hit._source.file_date || null`
     - `category: "wealth_signal"` (SEC filings are wealth indicators)
     - `relevance: "high"` (SEC filings are authoritative)
     - `raw_snippet: JSON.stringify(hit._source)`
   - Cache and return.
   - try/catch: return empty on error (never throw).

4. Export `searchEdgarEfts` via `withCircuitBreaker(searchEdgarEftsInternal, { name: "edgar-efts-channel", timeout: 10000 })`.

5. Register: `registerChannel("edgar-efts", searchEdgarEfts)`.
