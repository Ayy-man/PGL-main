   - Add "attom" for property/house/real-estate/owns queries
   - Return ONLY valid JSON: `{ "channels": [...], "reformulatedQuery": "...", "entityType": "person|company|property|general", "reasoning": "..." }`

   - Parse JSON from response text. If parse fails, return fallback: `{ channels: ["exa"], reformulatedQuery: query, entityType: "general", reasoning: "Parse failure, defaulting to exa" }`
   - Filter out channels whose API key is not configured: check `process.env.GNEWS_API_KEY` for gnews, `process.env.OPENCORPORATES_API_TOKEN` for opencorporates, `process.env.CRUNCHBASE_API_KEY` for crunchbase, `process.env.ATTOM_API_KEY` for attom.
