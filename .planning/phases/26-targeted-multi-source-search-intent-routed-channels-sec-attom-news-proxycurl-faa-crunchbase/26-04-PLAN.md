3. `RELEVANCE_SCORES` constant: `{ high: 3, medium: 2, low: 1 }`.

4. `CHANNEL_PRIORITY` constant — ordered list of channel preference for tie-breaking:
```typescript
const CHANNEL_PRIORITY: Record<string, number> = {
  "edgar-efts": 6,   // SEC filings most authoritative
  "exa": 5,          // Web intelligence, broad coverage
  "gnews": 4,        // News timeliness
  "crunchbase": 3,   // Company/funding data
  "opencorporates": 2, // Registration data
  "attom": 1,        // Property data (niche)
};
```

5. `mergeAndRank(results: ChannelResult[], query: string): ChannelResult[]`:
   - **Dedup by normalized URL:** Build `Map<string, ChannelResult>`. For duplicate URLs, keep the one with higher relevance score (or higher channel priority as tiebreak).
   - **Sort merged results by:**
     1. `RELEVANCE_SCORES[result.relevance]` descending
     2. `CHANNEL_PRIORITY[result.channelId]` descending (tiebreak)
     3. `event_date` descending if both have dates (newer first)
   - Return sorted array (all results, no limit — UI handles pagination)

6. Export `mergeAndRank`.

```typescript
export type ResearchResult = {
  results: ChannelResult[];

    resultCount: number;
    cached: boolean;
    latencyMs: number;
    error?: string;
  }>;

};
```

3. Define `ResearchParams` type:

```typescript
export type ResearchParams = {
  query: string;
  prospect: ChannelParams["prospect"];
  tenantId: string;
};
```

4. Implement `executeResearch(params: ResearchParams): Promise<ResearchResult>`:
   - Record `startMs = Date.now()`

   - **Latency optimization (per RESEARCH.md Open Question 1):** Fire Exa immediately in parallel with intent classification. Exa is always included, so we don't need to wait for the classifier to start it.

   ```typescript
   // Start Exa immediately (always fires) + classify intent in parallel
   const exaChannel = getChannel("exa");
   const exaPromise = exaChannel
     ? exaChannel({ query: params.query, prospect: params.prospect, tenantId: params.tenantId })
     : Promise.resolve({ channelId: "exa" as ChannelId, results: [], cached: false, latencyMs: 0, error: "Exa not registered" });

   const classifyPromise = classifyIntent(params.query, {
     name: params.prospect.full_name,
     company: params.prospect.company,
     title: params.prospect.title,
     publicly_traded_symbol: params.prospect.publicly_traded_symbol,
   });

   // Wait for classification
   const classification = await classifyPromise;
   ```

   - **Fire remaining channels in parallel:** For each channel in `classification.channels` EXCEPT "exa" (already started), get from CHANNEL_REGISTRY and fire.

   ```typescript
   const additionalChannelIds = classification.channels.filter(id => id !== "exa");
   const additionalPromises = additionalChannelIds.map(id => {
     const fn = getChannel(id);
     if (!fn) return Promise.resolve({ channelId: id, results: [], cached: false, latencyMs: 0, error: `Channel ${id} not registered` } as ChannelOutput);
     return fn({ query: classification.reformulatedQuery, prospect: params.prospect, tenantId: params.tenantId });
   });
   ```

   - **Collect all results via Promise.allSettled:**
   ```typescript
   const settled = await Promise.allSettled([exaPromise, ...additionalPromises]);
   ```

   - **Process settled results:** For each result:
     - If `fulfilled`: collect its `ChannelOutput`
     - If `rejected`: create a synthetic `ChannelOutput` with error
   - Build `channelStatuses` array from all outputs (including failed ones)
   - Collect all `ChannelResult[]` from successful outputs into a flat array

   - **Merge and rank:**
   ```typescript
   const mergedResults = mergeAndRank(allResults, params.query);
   ```

   - Return `ResearchResult` with results, classification, channelStatuses, totalLatencyMs.

5. Export `executeResearch`, `ResearchResult`, `ResearchParams`.
