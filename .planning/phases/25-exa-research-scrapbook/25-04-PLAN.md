    **State management (Claude's discretion — using useState + fetch, not useChat):**

    The AI SDK v6 `useChat` hook expects a specific message format. For this phase, use a simpler approach:
    - `messages: Array<{ role, content, cards?, metadata? }>` — local state
    - `isSearching: boolean` — loading state
    - `streamPhase: "idle" | "reasoning" | "tool" | "shimmer" | "cards" | "sources" | "complete"` — tracks current phase for UI
    - `currentCards: ScrapbookCard[]` — cards received so far for current response
    - `sessionId: string | null` — current session
    - `suggestions: string[]` — contextual suggestions (empty after first query)

    **Implementation approach:**
    Since the API route uses `createUIMessageStream` which streams AI SDK protocol, the simplest client approach is to use `fetch` with a ReadableStream reader and parse the SSE events. This avoids fighting with `useChat` expectations about message format.
