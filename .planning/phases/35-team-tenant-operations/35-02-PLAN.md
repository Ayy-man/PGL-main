# Plan 35-02: Seat Limits / Usage Quotas

## Goal

Add seat limits per tenant so invites are gated by plan capacity. Currently any tenant can invite unlimited users.

## Tasks

### Task 1: DB migration for tenant quotas
- Create `supabase/migrations/20260410_tenant_quotas.sql`
- Add columns to `tenants` table:
  - `max_seats INTEGER DEFAULT 5`
  - `plan TEXT DEFAULT 'starter'`
- Push migration to Supabase

### Task 2: Update TypeScript types
- Edit `src/types/database.ts` — add `max_seats` and `plan` to `Tenant` interface
- Update any zod schemas that validate tenant data

### Task 3: Enforce seat limit on invite
- Edit `src/app/actions/team.ts` — in `inviteTeamMember` (or equivalent):
  - Count current active users for the tenant (`SELECT count(*) FROM users WHERE tenant_id = ? AND is_active = true`)
  - Compare against `tenants.max_seats`
  - Reject with clear error: "Seat limit reached (X of Y seats used)"

### Task 4: Show seat usage on team page
- Edit `src/app/[orgId]/team/page.tsx`
- Add badge/indicator: "X of Y seats used" near the invite button
- If at limit, disable invite button with tooltip explaining why

### Task 5: Add quota fields to admin tenant creation
- Edit `src/app/admin/tenants/new/page.tsx` (or the create/edit tenant form)
- Add `max_seats` number input and `plan` dropdown (starter/professional/enterprise)
- Also add to tenant edit form if it exists

## Verification
- [ ] New tenants created with default max_seats=5
- [ ] Invite blocked when seat limit reached with clear error
- [ ] Team page shows "X of Y seats used"
- [ ] Super admin can set custom seat limits
