# Plan 35-04: Role Change by Tenant Admin

## Goal

Allow tenant admins to change a team member's role (agent <-> assistant) without needing to deactivate and re-invite.

## Tasks

### Task 1: Create changeUserRole server action
- Edit `src/app/actions/team.ts`
- Add `changeUserRole(userId, newRole)` server action
- Validate: caller must be `tenant_admin`, target user must be in same tenant
- Allowed transitions: `agent` <-> `assistant` only
- Reject attempts to set `tenant_admin` or `super_admin` (only super admins can do this)
- Update `public.users.role`
- Update `auth.users.app_metadata.role` via admin API: `supabase.auth.admin.updateUserById(userId, { app_metadata: { role: newRole } })`
- Log activity with `role_changed` or appropriate action type

### Task 2: Add role selector to team page
- Edit `src/app/[orgId]/team/page.tsx`
- In the actions column, add a role dropdown/select for each non-admin user
- Options: "Agent" | "Assistant"
- Current role pre-selected
- On change: call `changeUserRole` action
- Disabled for: the current user (can't change own role), tenant_admin users (can't demote)
- Show toast on success/failure

## Verification
- [ ] Tenant admin can change agent to assistant and vice versa
- [ ] Role change updates immediately in the team table
- [ ] Cannot change own role or promote to tenant_admin
- [ ] Auth metadata updated (user's next request uses new role)
