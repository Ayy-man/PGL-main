# Plan 35-03: Pending Invite Visibility

## Goal

Show pending invites on the team page so tenant admins can see who hasn't accepted yet, resend invites, or revoke them. Currently only completed users are visible.

## Tasks

### Task 1: Detect pending invite status
- In the team page data fetching, use admin client to check `auth.users` for each team member
- Users with `last_sign_in_at IS NULL` are pending invites
- Add a `status` field to the team member list: "active" | "pending" | "deactivated"

### Task 2: Display pending status in team table
- Edit `src/app/[orgId]/team/page.tsx`
- Show "Pending" badge (amber/yellow) next to users who haven't signed in
- Show "Active" badge (green) for active users
- Show "Deactivated" badge (red/gray) for deactivated users

### Task 3: Add resend invite action
- Edit `src/app/actions/team.ts`
- Add `resendInvite(userId)` server action
- Calls `supabase.auth.admin.inviteUserByEmail(email)` again
- Only available for pending users
- Rate limit: one resend per user per hour

### Task 4: Add revoke invite action
- Edit `src/app/actions/team.ts`
- Add `revokeInvite(userId)` server action
- Deletes both `public.users` row and `auth.users` record via admin API
- Confirmation required (use existing confirmation dialog pattern)
- Only available for pending users

### Task 5: Wire actions to team page UI
- Add "Resend" and "Revoke" action buttons in the actions column for pending users
- Show confirmation dialog before revoke
- Show toast on success/failure

## Verification
- [ ] Pending invites show "Pending" badge
- [ ] Resend invite sends new email
- [ ] Revoke removes user from both auth and public tables
- [ ] Revoke confirmation dialog works
- [ ] Active users don't show resend/revoke options
