# Plan 35-05: User Removal

## Goal

Add the ability to fully remove a user from a tenant (delete from both public.users and auth.users), not just deactivate them. Deactivated users still appear in the team list and their records persist.

## Tasks

### Task 1: Create removeTeamMember server action
- Edit `src/app/actions/team.ts`
- Add `removeTeamMember(userId)` server action
- Guards:
  - Cannot remove yourself
  - Cannot remove the last `tenant_admin` (count remaining active tenant_admins first)
  - Caller must be `tenant_admin`
  - Target must be in same tenant
- Delete `public.users` row
- Delete auth account via `supabase.auth.admin.deleteUser(userId)`
- Log activity before deletion (so the log entry captures who was removed)

### Task 2: Add remove button to team page
- Edit `src/app/[orgId]/team/page.tsx`
- Add "Remove" button (red/destructive variant) in actions column
- Wire to confirmation dialog (reuse existing `Confirmation` component or `AlertDialog`)
- Confirmation message: "Remove {name}? This cannot be undone."
- Disabled for self and last tenant_admin
- Show toast on success/failure

## Verification
- [ ] Remove button appears for non-self team members
- [ ] Confirmation dialog prevents accidental removal
- [ ] Removed user disappears from team list
- [ ] Removed user cannot log in
- [ ] Cannot remove last tenant_admin
