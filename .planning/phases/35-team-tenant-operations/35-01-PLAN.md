# Plan 35-01: Tenant Settings Page

## Goal

Give tenant admins a way to update their organization's name, slug, logo, and theme after initial onboarding. Currently these can only be changed via the super admin panel.

## Tasks

### Task 1: Create tenant settings server action
- Create `src/app/actions/tenant-settings.ts`
- `updateTenantSettings(formData)` — updates tenant name, slug, theme in `tenants` table
- Validate: slug format (lowercase alphanumeric + hyphens), slug uniqueness, name not empty
- Require `tenant_admin` role check
- Log activity with `tenant_settings_updated` action type
- Reuse validation patterns from `confirmTenantOnboarding` in `src/app/actions/onboarding.ts`

### Task 2: Create organization settings page
- Create `src/app/[orgId]/settings/organization/page.tsx`
- Form fields: org name (text input), slug (text input with preview), logo (reuse `LogoUpload` component), theme (reuse `ThemePicker` component)
- Wire to server action from Task 1
- Show success/error toasts
- RBAC gate: only render for `tenant_admin` role, redirect or show "unauthorized" for other roles

### Task 3: Add organization settings to nav
- Update nav items to show "Organization" as a sub-item under Settings for tenant admins
- Route: `/{orgId}/settings/organization`
- Only visible to `tenant_admin` role

## Verification
- [ ] Tenant admin can access org settings page
- [ ] Name/slug/theme/logo changes persist
- [ ] Agents/assistants cannot access the org settings page
- [ ] Activity logged for settings changes
