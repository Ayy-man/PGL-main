# Plan 15-03: Onboarding Page — Confirm Tenant Settings

## Goal
Build the `/onboarding/confirm-tenant` page where invited tenant admins confirm/edit their tenant's name, slug, colors, and set their password.

## Tasks

### Task 1: Create onboarding page
**File:** `src/app/onboarding/confirm-tenant/page.tsx` (new)

Client component page:
1. On mount, fetch current user session via Supabase client
2. Fetch tenant details from `/api/onboarding/tenant` (new endpoint)
3. Display form pre-filled with tenant data:
   - Tenant Name (editable)
   - Slug (editable, same pattern validation as create form)
   - Logo URL (editable)
   - Primary Color (editable)
   - Secondary Color (editable)
   - Divider
   - New Password (required, min 8 chars)
   - Confirm Password (required, must match)
4. Submit calls `confirmTenantOnboarding` server action
5. On success, redirect to `/{tenantSlug}/dashboard`

**Styling:**
- Centered card layout (max-w-lg mx-auto)
- surface-card background
- Gold heading: "Welcome to {tenantName}" in font-serif
- Subtitle: "Confirm your organization's settings to get started"
- Same form input styling as create tenant page
- Gold CTA submit button: "Confirm & Get Started"

### Task 2: Create onboarding API endpoint
**File:** `src/app/api/onboarding/tenant/route.ts` (new)

GET endpoint:
- Requires authenticated user
- Reads `tenant_id` from user's `app_metadata`
- Fetches tenant from DB using admin client
- Returns tenant data (name, slug, logo_url, primary_color, secondary_color)
- 403 if no tenant_id or `onboarding_completed !== false`

### Task 3: Create confirmTenantOnboarding server action
**File:** `src/app/actions/admin.ts`

New server action:
1. Validate session — user must have `onboarding_completed: false`
2. Validate form data (tenant name, slug, colors, password, confirm_password)
3. Update tenant record in DB (name, slug, logo_url, colors)
4. Update user password via `auth.admin.updateUserById(userId, { password })`
5. Set `app_metadata.onboarding_completed: true` via `auth.admin.updateUserById`
6. Log `tenant_confirmed` activity with metadata `{ tenant_name, changes: { before, after } }`
7. Log `user_invite_accepted` activity
8. Return tenant slug for redirect

### Task 4: Create onboarding layout
**File:** `src/app/onboarding/layout.tsx` (new)

Minimal layout:
- Dark background with ambient glow (same as root)
- No sidebar, no top bar — clean onboarding experience
- Centered content area
