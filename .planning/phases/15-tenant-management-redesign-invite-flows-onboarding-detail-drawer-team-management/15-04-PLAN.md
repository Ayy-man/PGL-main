# Plan 15-04: Tenant Detail API Endpoints

## Goal
Create API endpoints that power the tenant detail drawer with usage stats, health score, users, personas, and activity data.

## Tasks

### Task 1: Tenant details endpoint
**File:** `src/app/api/admin/tenants/[id]/route.ts` (new)

GET endpoint returning full tenant details:
- Tenant record (name, slug, logo, colors, is_active, created_at, updated_at)
- User count and list: `{ id, email, full_name, role, is_active, created_at, last_sign_in_at }`
  - Fetch from public.users where tenant_id matches
  - Get last_sign_in_at from auth.users via admin client
- Requires super_admin role (inline check per Phase 04 pattern)

### Task 2: Tenant usage stats endpoint
**File:** `src/app/api/admin/tenants/[id]/usage/route.ts` (new)

GET endpoint returning usage data:
- Query `usage_metrics_daily` for this tenant, last 30 days
- Aggregate totals: searches, profiles_viewed, profiles_enriched, csv_exports, logins
- Daily breakdown for sparkline data (array of { date, value } per metric)
- Previous 30-day totals for comparison (% change)
- Requires super_admin role

### Task 3: Tenant health score endpoint
**File:** `src/app/api/admin/tenants/[id]/health/route.ts` (new)

GET endpoint computing health score (0-100):
- **Activity frequency (30pts):** Days with activity in last 30 / 30 * 30
- **User engagement (25pts):** Active users (logged in last 7d) / total users * 25
- **Feature adoption (25pts):** Distinct action types used in last 30d / 11 * 25
- **Data freshness (20pts):** Based on days since last activity — 0d=20, 1-3d=15, 4-7d=10, 8-14d=5, 15+d=0
- Return: `{ score, breakdown: { activity, engagement, adoption, freshness }, status: 'healthy'|'warning'|'critical' }`
  - healthy >= 60, warning >= 30, critical < 30
- Requires super_admin role

### Task 4: Tenant top personas endpoint
**File:** `src/app/api/admin/tenants/[id]/personas/route.ts` (new)

GET endpoint returning top 3 personas:
- Query `personas` where tenant_id matches
- Order by `last_used_at` DESC NULLS LAST, limit 3
- Return: id, name, last_used_at, filters (for tag extraction)
- Requires super_admin role

### Task 5: Tenant activity endpoint
**File:** `src/app/api/admin/tenants/[id]/activity/route.ts` (new)

GET endpoint returning tenant-scoped activity:
- Query params: `tab` (all | admin | user), `page`, `limit` (default 5)
- `admin` tab filters: tenant_created, tenant_renamed, tenant_settings_updated, tenant_confirmed, user_invited, user_invite_accepted
- `user` tab filters: all other action types
- `all` tab: no filter
- Uses admin client with explicit tenant_id filter (not RLS — super_admin viewing cross-tenant)
- Return: `{ data, total, has_more }`
- Requires super_admin role
