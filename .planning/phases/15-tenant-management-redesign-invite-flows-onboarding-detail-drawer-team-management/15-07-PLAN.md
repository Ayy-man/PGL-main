# Plan 15-07: Team Management Page — Tenant Admin Invites Users

## Goal
Build the `/[orgId]/team` page where tenant admins can view their team members and invite new users via email.

## Tasks

### Task 1: Create team page
**File:** `src/app/[orgId]/team/page.tsx` (new)

Server component:
- Title: "Team" (font-serif)
- Subtitle: "Manage your team members and send invitations"
- Requires `tenant_admin` role (redirect to dashboard if agent/assistant)
- Fetch users for this tenant from DB (session client — RLS auto-scopes)
- Display users in a table: Name, Email, Role badge, Status badge, Last Active, Actions
- Actions column: status toggle (reuse pattern from admin users page)
- "Invite Team Member" button in top-right (opens invite dialog)

**Styling:**
- Same surface-admin-card table pattern used in admin pages
- admin-thead, admin-row-hover classes
- Gold CTA for invite button

### Task 2: Create invite dialog component
**File:** `src/app/[orgId]/team/invite-dialog.tsx` (new)

Client component using shadcn `Dialog`:
- Trigger: "Invite Team Member" button
- Form fields:
  - Email (required)
  - Full Name (optional — can be filled during onboarding)
  - Role: dropdown with `agent` and `assistant` only (no tenant_admin, no super_admin)
- Submit calls `inviteTeamMember` server action
- Success: close dialog, show toast/success message, revalidate page
- Error: show inline error message

### Task 3: Create inviteTeamMember server action
**File:** `src/app/actions/team.ts` (new)

Server action:
1. Validate session — require `tenant_admin` role
2. Get tenant_id from session user's app_metadata
3. Validate form data (email required, role must be agent or assistant)
4. Call `auth.admin.inviteUserByEmail(email, { data: { full_name }, redirectTo: '{origin}/api/auth/callback' })`
5. Set app_metadata: `{ role, tenant_id, onboarding_completed: false }`
6. Create public.users row: `{ email, full_name, role, tenant_id, is_active: true }`
7. Log `user_invited` activity with metadata: `{ email, role, invited_by: currentUser.id }`
8. Handle duplicate email error
9. Revalidate `/{orgId}/team`

### Task 4: Handle user invite acceptance for non-admin users
**File:** `src/app/api/auth/callback/route.ts`

Ensure the existing onboarding redirect (from Plan 15-01) works for regular users too:
- If `onboarding_completed: false` AND role is NOT `tenant_admin`:
  - Redirect to a simpler onboarding: `/onboarding/set-password`
  - This page only asks for password (no tenant settings)

### Task 5: Create simple user onboarding page
**File:** `src/app/onboarding/set-password/page.tsx` (new)

Simpler onboarding for non-admin invited users:
- Heading: "Welcome to {tenantName}" (font-serif)
- Subtitle: "Set your password to get started"
- Fields: New Password, Confirm Password
- Submit: update password, set `onboarding_completed: true`, log `user_invite_accepted`, redirect to `/{tenantSlug}/dashboard`
- Same centered card layout as confirm-tenant page

### Task 6: Add Team nav item to tenant sidebar
**File:** `src/components/layout/nav-items.ts` (or equivalent)

Add "Team" nav item:
- Icon: Users (from lucide-react)
- Path: `/{orgId}/team`
- Only visible to `tenant_admin` role
- Placed after existing nav items
