# Plan 15-02: Refactor Create Tenant + Admin Invite Flow

## Goal
Merge tenant creation and admin invitation into a single flow. Super admin creates a tenant and invites the first admin via email in one step.

## Tasks

### Task 1: Update createTenant server action
**File:** `src/app/actions/admin.ts`

Modify `createTenant` to accept an optional `admin_email` field in the FormData:
1. Add `admin_email` (optional email string) to `createTenantSchema`
2. After creating the tenant, if `admin_email` is provided:
   a. Create user in Supabase Auth via `auth.admin.inviteUserByEmail(email, { data: { full_name: '' }, redirectTo: '{origin}/api/auth/callback' })`
   b. Set `app_metadata`: `{ role: 'tenant_admin', tenant_id: tenant.id, onboarding_completed: false }`
   c. Create public.users row with `role: 'tenant_admin'`, `tenant_id: tenant.id`, `is_active: true`
   d. Log `tenant_created` activity with metadata `{ tenant_name: name, admin_email }`
   e. Log `user_invited` activity with metadata `{ email, role: 'tenant_admin', invited_by: currentUser.id }`
3. If tenant creation succeeds but invite fails, still return success but include a warning
4. Handle duplicate email errors gracefully

### Task 2: Update Create Tenant form
**File:** `src/app/admin/tenants/new/page.tsx`

Add an "Admin Email" field to the form:
- Label: "Admin Email (optional)" with helper text: "Send an invite to the tenant's first admin"
- Input type: email
- Placed after the slug field, before colors
- Same styling as other inputs (bg-input, rounded-[8px], gold focus ring)
- Show success message that includes "Invite sent to {email}" when admin_email was provided

### Task 3: Log tenant_created for tenants created without admin
**File:** `src/app/actions/admin.ts`

Ensure `tenant_created` activity is logged even when no admin email is provided, with metadata `{ tenant_name: name }`.
