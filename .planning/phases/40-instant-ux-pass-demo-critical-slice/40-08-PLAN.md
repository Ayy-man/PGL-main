```markdown
# Phase 40 — UAT

**Target:** https://pgl-main.vercel.app
**Run by:** (name) on (date)
**Result:** ☐ pass / ☐ fail

## Prerequisites
- Logged in as a tenant user with at least one saved search backed by active prospects
- Browser DevTools open (Network tab + Console)

## Steps

### 1. Realtime enrichment green-without-refresh (Adrian's literal complaint)
- **Do:** Open a list members page. Click the re-enrich action on one member.
- **Expect:** Within ~3–10s, the enrichment status pill transitions to "Enriched" WITHOUT pressing Cmd+R.
- **Result:** ☐ pass / ☐ fail

### 2. Bulk-enrich skeleton → live rows
- **Do:** From search, select 3–5 prospects and click "Enrich Selection" into a list.
- **Expect:** New rows appear on the list with shimmer-skeleton placeholders. Within ~5–30s, placeholders flip to real enriched data without refresh.
- **Result:** ☐ pass / ☐ fail

### 3. Create list — immediate appearance
- **Do:** Click "Create List", fill in a name, submit.
- **Expect:** The new list card appears in the grid IMMEDIATELY (with a dimmed / pending visual), reconciles to its normal appearance within ~500ms.
- **Result:** ☐ pass / ☐ fail

### 4. Delete list — immediate removal + undo
- **Do:** Click Delete on a list. Confirm.
- **Expect:** Card disappears from grid IMMEDIATELY. A toast with an "Undo" button appears for ~5s.
- **Do:** Click Undo within the toast window.
- **Expect:** Card reappears in its previous position.
- **Result:** ☐ pass / ☐ fail

### 5. Dismiss a search prospect — immediate removal
- **Do:** In search results, click dismiss on a prospect card.
- **Expect:** Card disappears IMMEDIATELY. Undo toast appears.
- **Do:** Click Undo.
- **Expect:** Card reappears.
- **Result:** ☐ pass / ☐ fail

### 6. Add/remove a tag — immediate toggle
- **Do:** On a prospect profile, add a new tag.
- **Expect:** Tag chip appears IMMEDIATELY. Stays after server round-trip.
- **Do:** Remove the tag.
- **Expect:** Chip disappears IMMEDIATELY.
- **Do:** (optional) Use DevTools to block the POST /tags request. Try adding a tag.
- **Expect:** Chip appears then disappears with a destructive toast showing the error.
- **Result:** ☐ pass / ☐ fail

### 7. Inline profile edit — rollback on bad input
- **Do:** On a prospect profile, edit an inline field (e.g., title) with intentionally invalid input — or use DevTools to block PATCH /profile.
- **Expect:** Field displays the new value briefly, then rolls back to the prior value with a destructive toast.
- **Result:** ☐ pass / ☐ fail

### 8. Polling fallback when WebSocket dies
- **Do:** Open DevTools Network tab. Filter to WS. Close/kill the open WebSocket connection.
- **Expect:** Within 10s, a `fetch` to the list endpoint (or `router.refresh` request) fires on a 10s interval.
- **Do:** Trigger a re-enrich on a list member.
- **Expect:** Green still eventually lands (within one poll interval after enrichment completes).
- **Result:** ☐ pass / ☐ fail

### 9. Channel cleanup — no ghost subscriptions
- **Do:** Navigate TO a list members page (channels subscribe), then navigate AWAY.
- **Expect:** DevTools Console shows no "channel not closed" warnings. React DevTools Profiler shows the channel effect's cleanup ran.
- **Alt verification:** `grep -c "removeChannel" src/` ≥ `grep -c "supabase.*\\.channel(" src/`.
- **Result:** ☐ pass / ☐ fail

## Sign-off
- Tester name: _______________
- Date: _______________
- Overall: ☐ ship / ☐ block

## Blockers (if any)
(list)
```
