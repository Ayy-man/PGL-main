  <action>
In `src/app/admin/admin-nav-links.tsx`, add a non-interactive section label above the nav items list.

1. Inside the `AdminNavLinks` component, before the `ADMIN_NAV.map(...)` block, add the section header:

```tsx
<div className="flex flex-col gap-1">
  {/* Section header — non-interactive label */}
  <div className="px-3 mb-1 mt-2">
    <p
      className="text-[10px] font-semibold uppercase tracking-widest"
      style={{ color: "var(--text-ghost)" }}
    >
      Platform Control
    </p>
  </div>

  {ADMIN_NAV.map((item) => {
    // ... existing nav link rendering
  })}
</div>
```

Key details:
- `text-[10px]` — smaller than nav item labels (13-14px), signals it's a category label not a link
- `font-semibold uppercase tracking-widest` — matches MASTER.md label typography spec
- `var(--text-ghost)` — `rgba(255,255,255,0.15)`, subtlest text tier
- `px-3` — aligns with nav item horizontal padding
- `mb-1 mt-2` — creates visual separation. The `mt-2` pushes below the sidebar header area, `mb-1` gives a small gap before the first nav item
- No `cursor-pointer` or hover handlers — this is purely a label, not interactive
- No `<Link>` or anchor tag — this is a `<div>` with a `<p>` inside

Do NOT:
- Add a "System Config" section — no routes exist for it (per research Open Question #3)
- Change any existing nav link styling, hover handlers, or active state logic
- Add dividers or horizontal rules between sections
