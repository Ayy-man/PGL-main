**Shared auth helper** (copy the `authenticate()` pattern from tags route):
```typescript
async function authenticate() {
  const supabase = await createClient();
  const { data: { user }, error: authError } = await supabase.auth.getUser();
  if (authError || !user) return { supabase, user: null, tenantId: null, authError: "Unauthorized" };
  const tenantId = user.app_metadata?.tenant_id as string | undefined;
  if (!tenantId) return { supabase, user, tenantId: null, authError: "Tenant ID not found" };
  return { supabase, user, tenantId, authError: null };
}
```

**GET handler:**
- Query params: `?category=outreach,data` (comma-separated filter), `?limit=50` (default 50, max 200), `?cursor=ISO_date` (for pagination by event_at)
- Use session client (RLS-scoped) to query `prospect_activity`
- Select: `id, prospect_id, tenant_id, user_id, category, event_type, title, note, metadata, event_at, created_at, triggers_status_change`
- Filter by prospect_id from URL params
- If `category` param present, filter with `.in('category', categories)`
- Order by `event_at` DESC
- If `cursor` param present, filter `.lt('event_at', cursor)` for keyset pagination
- Apply `.limit(parsedLimit)`
- Also join user info: do a second query to get user full_name for each unique user_id in results. Return as `users` map in response.
- Response shape: `{ events: ProspectActivity[], users: Record<string, { full_name: string }>, hasMore: boolean }`
- `hasMore` = events.length === limit (there might be more)
