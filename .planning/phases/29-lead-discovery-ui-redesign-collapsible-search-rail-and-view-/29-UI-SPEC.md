## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators, inline gaps |
| sm | 8px | Sidebar item vertical padding, compact element spacing |
| md | 16px | Sidebar item horizontal padding, card inner padding |
| lg | 24px | Section padding, tab content top padding |
| xl | 32px | Page-level vertical breathing room (Discover tab hero area) |
| 2xl | 48px | Discover tab h1-to-searchbar gap on desktop |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Sidebar expanded width: 240px (not a spacing token — structural dimension)
- Sidebar collapsed width: 48px (structural dimension; matches icon-only touch target minimum)
- Collapse toggle button: 44px × 44px minimum (WCAG touch target)
- Colored dot diameter: 8px (sidebar item dot indicator)
- Collapsed sidebar dot target area: 48px height per item (44px minimum met by row padding)
- Sidebar item gap: 12px — dense spacing between sidebar items (`gap-3`, `px-3`); matches existing `gap-3` Tailwind utility; structural density decision, not a layout spacing token. Sits between standard 8px and 16px tokens.

## Typography

Source: `globals.css` (`--font-sans: DM Sans`; `--font-serif: Cormorant Garamond`) and existing search-content.tsx patterns.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display (Discover tab h1) | 32px (sm: 38px) | 500 (font-medium, serif) | 1.2 | Cormorant Garamond |
| Heading (Saved search name as h1) | 28px (sm: 32px) | 500 (font-medium, serif) | 1.2 | Cormorant Garamond |
| Body / subtitle | 14px | 300 (font-light) | 1.5 | DM Sans |
| Label / UI chrome | 13px | 500 (font-medium) | 1.4 | DM Sans |

Exactly 2 weights across the design system: 300 (font-light) + 500 (font-medium).
- DM Sans: 300 (font-light, body/subtitle) + 500 (font-medium, labels/section headers/emphasis)
- Cormorant Garamond: 500 (font-medium) only — display use

Note: 13px + font-medium is the established token for all interactive pill/sidebar item text — carry unchanged from persona-pills.tsx.

## Layout Specifications

### Tab bar

```
[ Discover ]  [ Saved Searches ]
─────────────────────────────────   ← 1px var(--border-default) bottom rule
```

- Tab buttons: `text-[14px] font-medium px-4 py-3`
- Active tab: `color: var(--text-primary-ds)` + `2px solid var(--gold-primary)` bottom border
- Inactive tab: `color: var(--text-secondary-ds)`, no border
- Tab bar sits flush at the top of page content area (below the global TopBar)
- Tab switching: client-side state (`useState<'discover' | 'saved'>`), no route change

### Discover tab layout

```
┌─────────────────────────────────────────────┐
│  (pt-12 pb-8, max-w-2xl, mx-auto)           │
│                                             │
│  [h1: Display 32px serif]                  │
│  [subtitle: 14px font-light]               │
│                                             │
│  [NLSearchBar — full width]                │
│                                             │
│  ▼ Filters (collapsible, AdvancedFilters)  │
│                                             │
│  [Search]          [Save as new search]    │
│                                             │
│  ── Saved Searches ────────────────────────│  ← section divider
│  [shortcut list: compact, name + count]    │
└─────────────────────────────────────────────┘
```

- Discover tab content: `max-w-2xl mx-auto px-4 pt-12 pb-8`
- Action button row: `flex gap-3 mt-4`
- "Search" button: variant `outline` with gold border (`border: 1px solid var(--border-gold); color: var(--gold-primary)`)
- "Save as new search" button: variant `ghost`, `color: var(--text-secondary-ds)`
- Shortcut preview section: `mt-8`, section label `text-[13px] uppercase tracking-wider font-medium` with `color: var(--text-tertiary)`

### Saved Searches tab layout

```
┌────────────────────────────────────────────────────────┐
│ [SearchSidebarRail 240px] │ [Results panel flex-1]      │
│                           │                            │
│  ● Search Name 1          │  [SavedSearchViewHeader]   │
│  ● Search Name 2 (active) │  [ProspectResultsTable]    │
│  ● Search Name 3          │  [BulkActionsBar]          │
│                           │                            │
│  [+ New]                  │                            │
└────────────────────────────────────────────────────────┘
```

- Layout: `flex flex-row h-full`
- Sidebar: `flex-shrink-0`, width transitions between `240px` (expanded) and `48px` (collapsed) via CSS `transition: width 200ms ease`
- Results panel: `flex-1 overflow-y-auto px-6 py-6`

### Sidebar rail (expanded)

- Item height: `min-h-[44px]` (WCAG touch target)
- Item padding: `px-4 py-2`
- Item layout: `flex items-center gap-3` (gap-3 = 12px; structural density exception — see Spacing Exceptions)
- Dot: `w-2 h-2 rounded-full flex-shrink-0` with `getPersonaColor(id)` background
- Name text: `text-[13px] font-medium truncate`
- Collapse toggle: chevron button `h-9 w-9` at top of sidebar; `ChevronLeft` icon when expanded, `ChevronRight` when collapsed
  - `aria-label="Collapse sidebar"` when expanded
  - `aria-label="Expand sidebar"` when collapsed
- "+ New" button: dashed border pill at bottom, `text-[13px] font-medium`, same pattern as current PersonaPills create button

### Sidebar rail (collapsed — 48px)

- Only colored dot visible per item, centered horizontally
- Item layout: `flex items-center justify-center h-[44px]`
- Dot: same `getPersonaColor(id)` color, same 8px diameter
- Tooltip on hover: shadcn `Tooltip` component, content = full search name, `side="right"`
- Collapse toggle: same chevron button, now shows `ChevronRight` icon, positioned at top center of the 48px strip
  - `aria-label="Expand sidebar"` when collapsed

### View header (Saved Searches tab — search selected)

```
[Search Name]                       ← h1, serif 28–32px font-medium
{N} prospects • Last refreshed: Xh ago  [↺]   ← subtitle 14px font-light
```

- h1: `font-serif text-[28px] sm:text-[32px] font-medium` with `letterSpacing: "-0.5px"`, `color: var(--text-primary-ds)`
- Subtitle: `text-[14px] font-light mt-1`, `color: var(--text-tertiary)`
- Refresh icon: `RefreshCw` 14px, `cursor-pointer`, inline `ml-2`; on hover: `color: var(--gold-primary)`
- The subtitle is a single line with `•` separator (middot U+2022)

## Interaction Contracts

### Tab switching

| Trigger | Result |
|---------|--------|
| Click "Discover" tab | Active tab = discover; NL bar, filters, shortcut list visible |
| Click "Saved Searches" tab | Active tab = saved; sidebar + results panel visible |
| Submit NL search | Navigate to saved tab, results shown in right panel, no sidebar item highlighted |
| Submit search with named saved search | Navigate to saved tab, that item highlighted in sidebar |
| Click shortcut item on Discover tab | Navigate to saved tab, clicked search selected |

### Sidebar collapse

| Trigger | Result |
|---------|--------|
| Click collapse toggle (chevron) | Width animates from 240px to 48px (200ms ease); text hidden, dots only shown |
| Click expand toggle (chevron) | Width animates from 48px to 240px (200ms ease); text revealed |
| Hover over dot in collapsed state | shadcn Tooltip appears to the right with full search name |
| Click dot in collapsed state | Selects that saved search, shows results in right panel |

Collapse state persistence: store in `localStorage` key `pgl:search-sidebar-collapsed` (boolean). Restore on mount.

### Empty states

| State | Component | Content |
|-------|-----------|---------|
| Saved Searches tab, no searches exist | EmptyState (existing pattern) | See Copywriting Contract below |
| Saved Searches tab, searches exist but none selected | Inline prompt in results panel | See Copywriting Contract below |
| Discover tab, shortcut list is empty | Section label hidden; no shortcut list rendered | No empty message needed — section simply absent |

### Transition between tabs

- No route change; use `opacity` fade: outgoing content fades to 0 (100ms), incoming fades to 1 (200ms)
- Use the existing `page-enter` utility class (`fadeIn 0.4s ease forwards`) on tab content panels

## Saved Search Shortcut List (Discover Tab) — Visual Density

Format: compact list (not cards). Each row is `min-h-[40px]`, one row per saved search.

```
● Tech Execs             · 47 prospects    →
● Finance Elite          · 120 prospects   →
● BigLaw Partners        · 23 prospects    →
```

- Row layout: `flex items-center gap-3 px-3 py-2 rounded-[8px] cursor-pointer` (gap-3 = 12px; structural density exception — see Spacing Exceptions)
- Dot: `w-2 h-2 rounded-full` using `getPersonaColor(id)` — identical to sidebar
- Name: `text-[13px] font-medium`, `color: var(--text-primary-ds)`
- Count: `text-[13px]`, `color: var(--text-tertiary)`, preceded by `·` separator
- Arrow: `ChevronRight` 14px, `color: var(--text-ghost)`, `ml-auto`
- Hover: `background: var(--gold-bg); border: 1px solid var(--border-hover)`
- Show maximum 5 items; if more, show "View all {N} →" link in `text-[13px]` gold
