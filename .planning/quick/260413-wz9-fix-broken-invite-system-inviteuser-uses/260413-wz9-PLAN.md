**In `src/app/actions/team.ts`, rewrite `resendInvite()` (lines 209-244):**

Replace `inviteUserByEmail()` call with `generateLink()` which works for already-registered users:

1. After fetching `targetUser.email`, call:
   ```typescript
   const { data: linkData, error: linkError } = await admin.auth.admin.generateLink({
     type: 'magiclink',
     email: targetUser.email,
     options: { redirectTo }
   });
   ```
2. If `linkError`, return `{ error: linkError.message }`.
3. Note: `generateLink` generates the link server-side. Supabase automatically sends the email when using the admin API `generateLink` with type `magiclink`. If the email is NOT automatically sent by `generateLink`, fall back to `inviteUserByEmail` wrapped in a try-catch that catches "already registered" and then retries with `generateLink({ type: 'magiclink' })`.

Actually, the most reliable approach for resend: Use `admin.auth.admin.generateLink({ type: 'invite', email, options: { redirectTo } })`. The `invite` type works for re-inviting existing unconfirmed users. If Supabase returns an error because the user is already confirmed, catch it and use `generateLink({ type: 'magiclink' })` as fallback. This two-step approach covers both pending (unconfirmed) and confirmed-but-never-logged-in users.
