# Quick Task 260407-02n: Fix Activity Log Showing Empty — Summary

**One-liner:** Fixed empty activity log by switching GET /api/activity reads to admin client (bypassing RLS), adding activity_log migration, and wiring logActivity into Apollo search and add-to-list routes.

## What Was Done

### Task 1: Migration + API Route Fix

**Problem:** Three compounding root causes made the Activity Log page show nothing:
1. No migration file for `activity_log` — table existed only as a manual Supabase dashboard creation, fragile and undocumented.
2. RLS blocks reads — `logActivity()` writes via admin client (bypasses RLS), but `GET /api/activity` read via the session client (subject to RLS). With no SELECT policy on `activity_log`, the session client returned 0 rows.
3. High-frequency actions (search, add-to-list) had no `logActivity()` calls, so there was nothing to show even if reads worked.

**Fix:**
- Created `supabase/migrations/20260407_activity_log_table.sql` with `CREATE TABLE IF NOT EXISTS` (safe for existing prod DB), four indexes, and `ALTER TABLE ... ENABLE ROW LEVEL SECURITY`. RLS policies left to the Supabase dashboard per project convention.
- In `GET /api/activity`: imported `createAdminClient`, extracted `tenantId` from `user.app_metadata?.tenant_id` (403 if missing), replaced the session-client query with an admin-client query that includes an explicit `.eq("tenant_id", tenantId)` filter to manually scope results per tenant.

### Task 2: Add logActivity Calls

**Apollo search (`POST /api/search/apollo`):**
- Added `import { logActivity }` at the top.
- After `searchApollo()` returns successfully, fire-and-forget `logActivity({ actionType: "search_executed", ... })` with `resultCount`, `totalResults`, `cached`, `searchIdentifier`, and `page` in metadata.
- `.catch(() => {})` prevents unhandled rejection warnings.

**Add-to-list (`POST /api/prospects/add-to-list`):**
- Added `import { logActivity }` at the top.
- After computing `successCount`, if `successCount > 0`, fire-and-forget `logActivity({ actionType: "add_to_list", ... })` with `listIds` and `successCount` in metadata.
- Only logs on success — keeps the activity log clean.
