# Quick Task 260328-02d: Add Market Intelligence as 5th Auto-Triggered Enrichment Source

**One-liner:** Integrated Finnhub + Yahoo Finance market snapshot into the Inngest enrichment pipeline as Step 4.5, skipped automatically for private companies, with all 5 UI components updated to show the market source.

## What Was Done

Previously, market data was only available via manual refresh on the prospect profile page (POST `/api/prospects/[prospectId]/market-data`). This plan integrates it into the automated Inngest enrichment workflow.

## Tasks Completed

### Task 1: Extract shared market-data module + slim API route + update event types

**Commit:** `e859548`

- Created `src/lib/enrichment/market-data.ts` exporting `fetchMarketSnapshot(ticker, insiderData?)` and `InsiderData` interface
- The shared function contains all Finnhub + Yahoo Finance fetch/compute logic (no DB writes, no auth)
- Refactored `market-data/route.ts` to import and call `fetchMarketSnapshot` — route is now ~50% smaller
- Added `ticker?: string` to `"prospect/enrich.requested"` event type in `src/inngest/types.ts`
- Passed `ticker: prospect.publicly_traded_symbol || undefined` in the `inngest.send()` call in the enrich route

### Task 2: Add market step to Inngest pipeline + initialize market source status

**Commit:** `fb3cc5e`

- Destructured `ticker` from `event.data` in `enrich-prospect.ts`
- Replaced hardcoded 4-source status object with conditional initialization:
  - Public company with ticker: market = `{ status: "pending" }`
  - Private company or no ticker: market = `{ status: "skipped" }`
- Added Step 4.5 `fetch-market-data` between EDGAR and Claude AI:
  - Uses dynamic import (`await import("@/lib/enrichment/market-data")`) to avoid loading module at startup
  - Passes EDGAR insider transactions to `fetchMarketSnapshot` for equity estimation
  - Writes `stock_snapshot` + `stock_snapshot_at` to DB on success
  - Updates `enrichment_source_status.market` on complete/failed
  - Returns `{ found: false, status: "skipped" }` for private companies without re-marking (already set in step 1)
- Added `market: marketData.status` to finalize step activity log metadata and return value

### Task 3: Add "market" source to all 5 UI enrichment components

**Commit:** `0198054`

- **enrichment-status-dots.tsx**: Added `"market"` to `SOURCE_ORDER` array and `market: "Market Data"` to `SOURCE_LABELS` — now shows 5 dots for prospects with market source
- **enrichment-status.tsx**: Added `market: "Market Data"` to `SOURCE_LABELS`, changed grid to `flex flex-wrap gap-4` to accommodate 5 items
- **enrichment-tab.tsx**: Added `"market"` to `SOURCE_KEYS` + `SOURCE_LABELS`, changed grid to `flex flex-wrap gap-4` with cards having `flex-1 min-w-[140px]`
- **prospect-slide-over.tsx**: Added `market: "Market Data"` to `SOURCE_LABELS` (iterates `Object.entries(sourceStatus)` dynamically, so market auto-renders when present)
- **profile-view.tsx**: Replaced hardcoded `["contactout", "exa", "sec", "claude"]` with dynamic array that conditionally includes `"market"` based on `prospect.publicly_traded_symbol`; changed to `flex flex-wrap` with `flex-1 min-w-[calc(50%-3px)]` for clean 3+2 layout at 5 sources
